# Diary Writer

## 触发条件

- 触发词：「保存到日记」「保存链接」「记个待办」「有个想法帮我记一下」「帮我记一下」「保存一下」「帮我记录」
- 裸链接（无其他聊天意图）
- 任何保存信息到日记的指令

## 配置：日记路径

- 路径存储在 `skills/Diary-Writer/.diary_config`（纯文本，一行绝对路径，无末尾斜杠）
- 首次使用（`.diary_config` 不存在）→ 用 `grep_search` 搜索 `references/first-use-guide.md` 查「首次使用」段落，按其脚本引导用户设置路径
- 路径变更：用户说「修改日记路径」→ 询问新路径 → 覆盖 `.diary_config`

## 工作流

### 第一步：分类

| 类型  | 名称    | 识别特征                                           |
| --- | ----- | ---------------------------------------------- |
| 1   | 工作待办  | 关键词：文档、开会、跟进、PPT、规划、评审、方案、报告、需求、上线、发布、协调、对接、项目 |
| 2   | 个人待办  | 生活琐事+个人成长：买东西、预约、学、读、旅游、整理、锻炼、课程、公众号、书、考试      |
| 3   | 知识/链接 | 有URL，或>200字无任务关键词的长文本                          |
| 4   | 个人思考  | 想法、灵感、反思——无URL、无任务关键词、第一人称口吻、旁观者视角             |

**优先级**：有URL → 类型3。有任务关键词 → 类型1或2。个人反思性文字 → 类型4。
**模糊时**：无法判断类型1还是2 → 问「这是工作相关的还是个人的？」

### 第二步：确保日记文件存在

1. 读 `.diary_config` 获取路径
2. 计算目标文件：`{路径}/YYYY-MM-DD.md`
3. `read_file` 检查文件是否存在
4. 不存在 → 用 `read_file` 读 `examples/diary-template.md` 获取模板 → `write_file` 创建
5. 存在 → `read_file` 读全文，识别分区锚点

### 第三步：按类型执行

**所有类型**：先用 `grep_search`（pattern="对应类型名"，path="skills/Diary-Writer/references/format-guide.md"）定位该类型的格式规范和规则，再执行写入。

- **类型1（工作待办）**→ 写入 `## ⛳工作待办`
- **类型2（个人待办）**→ 写入 `## 🐱生活待办`
- **类型3（知识/链接）**→ 写入 `## 💩看了什么`。提炼知识要点前，必须 `read_file skills/Diary-Writer/references/hawkeye.md` 并逐条遵循。
- **类型4（个人思考）**→ 写入 `## 🤫想了什么`

### 第四步：写入日记

- **新日记**：`write_file`（完整模板）
- **已有日记**：`read_file` → `edit_file`（用分区标题做锚点追加）

## 批量处理

- 多条内容：逐条处理，全部完成后统一回复
- 混合类型：各自分类，按顺序写入对应分区
- > 3条：附带进度提示
- 多个URL：逐个提取（tavily_extract → 提炼 → 写入 → 下一条）

## 跳过条件

- 链接需登录 → 跳过，告知「链接需登录，已跳过」
- 链接返回空/错误 → 跳过，告知「链接无法读取，已跳过」
- 内容已在今日日记中 → 跳过，告知「已在今日日记中」

## 文件操作

- 日记文件：始终用 `.diary_config` 中的绝对路径 + `YYYY-MM-DD.md`
- `.diary_config`：纯文本，一行，绝对路径，无末尾斜杠
