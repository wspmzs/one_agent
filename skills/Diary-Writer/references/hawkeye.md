# 知识萃取铁律

> 执行知识萃取前必读。
> 作用域：仅影响 `💩看了什么` 中「高维思考要点」的输出。

## 萃取前快速清单

逐条过一遍再动笔。每条知识要点都必须经过这四问：

1. **作者默认了什么前提没说出来？**
2. **这个现象背后的底层机制是什么？**
3. **脱离这篇文章的语境，这个规律在什么场景下也成立？**
4. **如果反着看，结论会怎样？**

## 铁律

1. **零术语继承。** 禁止出现来源文章的任何名词、概念标签、专有术语、缩写、框架名、角色名。全部替换为通用描述。

2. **全文通用化。** 每条结论锚定在无需任何背景即可识别的生活或工作现象上，而非来源文章的概念体系。读者看不出触发了哪篇文章。

3. **独立成篇。** 任意抽取一条单独阅读，完全自洽。不依赖同节其他条目、不依赖摘要、不依赖文章标题。

4. **推文标准。** 每条放入微博或推特可直接发布。结论前置，逻辑内嵌。无论证痕迹（禁止「因为…所以…」式展开）。

## 禁忌

- 禁止以「这个模型」「这个框架」「这种思路」起句
- 禁止出现「文章指出」「作者认为」「文中提到」等引用标记
- 禁止关联用户过往言论、工作区记忆。
- 禁止表情符号。

## 思考维度

以下6个维度是萃取时的强制切入视角。至少触发3个。零触发等于原文复述 → 重写。

**第一性原理。** 剥离文章中的所有假设和既有框架，追问「这个结论成立的最底层前提是什么？」「原文的论证建立在哪些未经检验的默认前提之上？」——找到前提，才能判断结论的坚固程度。

**假设反转。** 翻转原文的因果关系（如果结果变成原因）、功能定位（如果核心技术成为副作用）、默认前提（如果公认假设被证伪），结论会如何重构？不是简单地「反过来想」，而是找到原文没有探索的分支。

**跨域映射。** 原文机制脱离当前语境，能否映射到另一完全不同的场景？找同构因果结构。同一个底层规律在不同领域的表现形式往往比原文本身更有价值。

**二阶效应。** 原文讨论的变化发生后，会引发什么连锁反应？追问「变化之上的变化」。真正的智慧不在第一层结果，而在第二层、第三层的系统传导。

**成本边界。** 方案的成本结构是什么？关键成本涨十倍还成立吗？原文避开了什么成本？——任何不讨论成本的方案都是不完整的。

**升维审视。** 将问题置于更大的系统、更长的时间轴中观察。如果尺度放大十倍（时间、空间、影响范围），这个结论还成立吗？会衍生出什么新的约束？——很多看似正确的结论，换个尺度就失效了。

## 输出格式

- 数字序号 `1.` 起头，一条一个结论。不合并。
- 短句。结论前置。
- 精简不丢细节。方法参数、操作路径等具象信息保留，拆为短句。
- 不限制字数，确保表达清晰、逻辑自洽。
- 不限制知识萃取条数，务必覆盖所有重要信息。
