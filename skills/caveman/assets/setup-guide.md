# Caveman 首次配置引导

> 本文件仅在检测到配置缺失时按需读取，不要在每次会话中加载。

---

## 两种压缩模式

| 模式 | 效果 | 适用场景 |
|------|------|----------|
| **FULL** | 去冠词、去连接词、碎片化短句，每句 2-5 词。压缩率 30%-75% | 长文本、跨上下文传输、极致省 token |
| **LITE** | 仅去冠词和冗余修饰，保留连接词与代词，句子上限 8 词 | 需保留可读性的场景 |

## 两种压缩方法

| 方法 | 原理 | 成本 | 触发方式 |
|------|------|------|----------|
| **路径 A — 会话内压缩** | Agent 直接采用 caveman 风格说话，无需外部调用 | 零成本，零延迟 | 用户说「caveman模式」「少废话」「说重点」等 |
| **路径 B — 外部工具压缩** | 调用 Python 脚本，经 LLM API 压缩 + 可选向量相似度校验 | 消耗 API 调用费用 | 用户说「调工具压缩」或提供文件并要求压缩 |

---

## 配置步骤（仅路径 B 需要）

### 第 1 步：编辑 `config.json`

路径：`skills/caveman/config.json`，按你的 API 服务填写：

```json
{
  "压缩模型": {
    "base_url": "https://api.deepseek.com",
    "model_id": "deepseek-v4-flash"
  },
  "向量比对模型": {
    "base_url": "http://127.0.0.1:11434/v1",
    "model_id": "qwen3-embedding:4b"
  }
}
```

- `压缩模型`：执行文本压缩的 LLM（路径 B 必填）
- `向量比对模型`：压缩后相似度校验的 Embedding 模型（仅 `-v` 参数时使用，非必填）
- `相似度验证.阈值`：PASS/WARN 分界线，默认 `0.75`，可直接改 `config.json`

### 第 2 步：设置环境变量（API 密钥）

密钥不存放在文件中，必须通过环境变量配置：

| 环境变量名 | 用途 | 必填 |
|-----------|------|------|
| `CAVEMAN_MODEL_KEY` | 压缩模型的 API 密钥 | ✅ 使用路径 B 必填 |
| `CAVEMAN_EMBEDDING_KEY` | 向量比对模型的 API 密钥 | 仅使用 `-v` 验证时必填 |

**设置方法**：QwenPaw → 设置 → 环境变量 → 添加 → Name 填变量名，Key 填你的 API 密钥值。

> 压缩模型和向量比对模型使用同一 API 服务时，两个环境变量填相同值即可。

### 验证配置

```bash
cd skills/caveman/scripts
python caveman_compress.py "测试文本，验证压缩功能" -i full
```

预期输出：
```
[CAVEMAN FULL]
测试文本验证压缩功能
```
