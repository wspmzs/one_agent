## 这个 SKILL 能干什么

把**模糊的想法**变成**可执行的结构化文档**。适用场景包括：

- 产品需求文档（PRD）
- 技术方案设计
- 旅行计划、学习计划
- 装修方案、活动策划
- 任何需要「想清楚再写下来」的场景

你只需要把脑子里的东西倒出来，agent 会扮演引导者，帮你澄清矛盾、补全缺失、逐节起草、迭代打磨。

## 工作流概览

```
阶段一                阶段二                  阶段三
需求理解与澄清  ──→  方案精炼与结构化  ──→  可实施性验证与盲点检测
```

### 阶段一：需求理解与澄清

- 收集元信息（文档类型、受众、模板、约束）
- 鼓励你信息倾泻（不限格式）
- 提出 5-10 个澄清问题，揪出逻辑矛盾、缺失信息、隐含假设

### 阶段二：方案精炼与结构化

- 确定文档结构 → 创建骨架文件
- 逐节推进，每节走 6 步：
  1. **澄清焦点** — 对该节提问
  2. **头脑风暴** — 列出可选要点
  3. **筛选与组合** — 你勾选保留哪些
  4. **缺口检查** — 有没有遗漏
  5. **起草** — agent 写，你不用动手
  6. **迭代打磨** — 你说怎么改，agent 执行
- 全部章节完成后整体审阅

### 阶段三：可实施性验证

- 将文档交给另一个 agent（如小美）做「零上下文」审查
- 模拟真实读者视角，找出模糊点、隐藏假设、矛盾
- 根据发现回到阶段二修复
