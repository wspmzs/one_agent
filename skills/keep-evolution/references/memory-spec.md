# MEMORY.md 写入规范

> **映射文件：** MEMORY.md（各Agent工作区根目录）
> **核心原则：** 启动文件必须自包含。规则写进文件，不依赖外部引用。

## 定位

长期可复用经验库。不含：待办事项（放日志）、写入规范本身（放本规范文件）、实时状态（会过期）。

## 元数据

头部必须包含元数据：

```yaml
---
summary: "长期可复用经验与知识库"
read_when:
  - 手动引导工作区
  - 每次会话启动时
---
```

## 章节结构

### 1. 工作流程（SOP）

**【强约束】必须写：**

- 常用操作序列：重复执行的实践步骤
- 命令速查：常用命令、参数组合
- 触发规则：什么条件触发什么流程

**【格式建议】** 命令用代码块，步骤用编号列表。标题+说明+代码块。

**【示例】**

```
### Skill创建流程
1. 读skill-creator/SKILL.md
2. 创建skills/<name>/目录
3. 编写SKILL.md + scripts/
4. 测试 → 更新配置中的skills列表
```

**【与AGENTS.md的SOP区分】**

- AGENTS.md写"规范流程"：Agent应该遵循的标准工作流程
- MEMORY.md写"实践步骤"：Agent实际执行过的、可复用的操作序列

---

### 2. 经验和教训

**【强约束】必须写：**

- 方法论总结：从多次任务抽象的通用原则
- 踩坑记录：坑+根因+标准解法
- 用户工作偏好：从互动沉淀的用户习惯

**【格式建议】** 每条 `###` 三级标题，正文3-5行。整段叙述。

**【示例】**

```
### 循环依赖检测（2026-05-13）
同批创建的Skill可能互相引用不存在的工具。解法：创建前扫描所有引用，修复优先级：自验证>外部依赖可选>平台兼容。

### 用户能力画像纠正（2026-05-11）
能讨论技术原理≠能写代码。首次建立画像时问清职业+技术边界，不默认推导。
```

**【与PROFILE.md的区分】**

- PROFILE.md写"用户偏好"：用户希望怎么被对待
- MEMORY.md写"用户工作偏好"：从实际互动中沉淀的习惯（更具体、更实践）

---

### 3. 领域知识

**【强约束】必须写：**

- 配置备忘：工具配置格式、API Key位置、环境变量
- 架构信息：系统拓扑、关键路径
- 工具列表：已安装工具/插件版本、入口路径

**【格式建议】** 配置用代码块（JSON/YAML），架构用表格。引用外部文档用路径链接。

**【示例】**

```
### 工具配置格式示例
```json
{"name": "显示名", "enabled": true, "transport": "stdio", "command": "npx", "args": ["-y", "package"]}
```
```

---

### 4. 专有名词解释

**【强约束】必须写：**

- 内部术语：团队/项目自造词定义
- 缩写全称：常用缩写展开
- 命名约定：文件/目录/Agent命名规则

**【格式建议】** `- **术语：** 一句话定义`。按字母或领域分组。

**【示例】**

```
### 内部术语
- **临时工沙盒模式：** 非核心Skill先在临时Agent验证，好用迁移/不好丢弃
- **精简压缩：** Agent间通信的token压缩协议，降65-75%
```

---

### 5. 项目经历

**【建议】可选写（无内容可删除章节）：**

- 项目名称、时间、角色、关键成果
- 复用经验、避坑指南

**【格式建议】** `### 项目名（日期）`，正文3-5行。

**【与PROFILE.md的区分】**

- PROFILE.md写"当前项目参与"：项目名、角色、状态（列表形式）
- MEMORY.md写"项目经历"：详细的经验教训、方法论总结（叙述形式）

---

## 强约束（不写什么）

**绝对不写：**

- 待办事项（放日志）
- 写入规范本身（放本规范文件）
- 实时状态跟踪（会过期，改"当前可用"或"已弃用"）
- Agent身份、行为准则（归SOUL.md）
- 用户画像、偏好（归PROFILE.md）
- 外部文件引用链

**格式限制：**

- 纯Markdown
- 不用流程图
- 不用mermaid图、ASCII码
- 命令类用代码块（代码块不超过5行，超过时拆分或简化）
- 历史条目加日期标注
- 不使用表情
