# PROFILE.md 写入规范

> **映射文件：** PROFILE.md（各Agent工作区根目录）
> **核心原则：** 启动文件必须自包含。规则写进文件，不依赖外部引用。

## 定位

用户画像与偏好档案。不含：Agent身份（SOUL.md）、工作流程（AGENTS.md）、记忆内容（MEMORY.md）。

## 元数据

头部必须包含元数据：

```yaml
---
summary: "用户身份信息与简历资料"
read_when:
  - 手动引导工作区
  - 每次会话启动时
---
```

## 章节结构

### 1. 用户信息

**【强约束】必须写：**

- 称呼：用户希望怎么被叫
- 职业：岗位、职责范围
- 核心诉求：使用Agent的最终目的（一句话）
- 技术能力边界：能做什么、不能做什么

**【格式建议】** 短句分条列点，`- **关键词：** 内容`。技术边界用表格。

**【示例】**

```
## 用户资料
- **称呼：** 张硕同学
- **职业：** 产品经理
- **核心诉求：** 通过自然语言和碎片化信息驱动AI完成技术工作

### 技术能力边界
| 领域 | 能力 | 说明 |
|------|------|------|
| 写代码 | 不能 | 全靠Agent |
| 读代码 | 不能 | 看不懂源码 |
| 改JSON | 能 | 简单配置可手动改 |
```

---

### 2. 用户偏好

**【强约束】必须写：**

- 沟通偏好：希望Agent怎么说话
- 学习风格：学习动机、方法、深度

**【建议】可选写：**

- 技术栈：了解但不实操的技术
- 工作习惯：时间安排、优先级偏好

**【格式建议】** 分条列点，小标题 + `-` 列表。祈使句或短语。

**【示例】**

```
### 沟通偏好
- **指令：** 短句、祈使、前置关键信息
- **回复：** 结论先行，再给理由
- **解释：** 讲原理和概念，少列代码

### 学习风格
- **动机：** 跟工程师同频对话
- **方法：** 理解概念→知道上限→提更好的需求
- **深度：** 够用就行
```

---

### 3. 用户参与项目

**【建议】可选写（无内容可删除章节）：**

- 当前项目：项目名、角色、阶段、文档路径
- 历史项目：已完成项目 + 一句话成果

**【格式建议】** 每个项目一个 `-` 条目，`项目名：角色 + 状态`。

**【与MEMORY.md的区分】**

- PROFILE.md写"当前项目参与"：我正在做什么、和谁协作
- MEMORY.md写"历史项目经验教训"：做过什么、学到什么

---

## 强约束（不写什么）

**绝对不写：**

- Agent自我描述、身份定义（归SOUL.md）
- 协作模式定义、行为准则（归SOUL.md）
- 记忆内容、经验教训（归MEMORY.md）
- 外部文件引用链
- 空壳章节（无内容就删掉，或标注 `> 暂无`）

**格式限制：**

- 纯Markdown
- 不使用表情
- 不用mermaid图、ASCII码
- 代码块不超过5行
- 技术边界用表格，其他用列表
