# Self-Review Checklist

Use this when reviewing output. One per deliverable.

## Output: [describe what this is]

### Check 1: Goal Match

- [ ] User's original request: [write it here]
- [ ] This output does exactly that
- [ ] If not: what changed and why

### Check 2: Security

- [ ] No API keys visible
- [ ] No passwords or tokens
- [ ] No private paths
- [ ] No internal URLs

### Check 3: Code Validity

- [ ] Code is real, not pseudo
- [ ] Syntax is correct
- [ ] Imports/dependencies listed
- [ ] Paths are portable

### Check 4: Completeness

- [ ] All promised content present
- [ ] No TODO/FIXME/PLACEHOLDER left
- [ ] No unfinished sentences

### Check 5: Format

- [ ] Markdown renders correctly
- [ ] Code blocks have language tags
- [ ] Tables are aligned
- [ ] Links are valid

## Result

- [ ] ALL CHECKS PASSED → Deliver
- [ ] ISSUES FOUND → Fix then re-check
